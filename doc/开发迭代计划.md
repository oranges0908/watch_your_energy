# WatchYourEnergy — 分步开发迭代计划

> 每个迭代包含：设计确认 → 开发 → 验证 三个完整环节。
> 前一迭代的验证通过，才进入下一迭代。

---

## 迭代总览

| # | 名称 | 核心产出 | 依赖 |
|---|---|---|---|
| I1 | 基础设施 | DB Schema + Pydantic模型 + 项目骨架 | — |
| I2 | Step生成引擎 | validators + agent loop + fallback | I1 |
| I3 | 后端API层 | 所有 repo/service/router + FastAPI完整运行 | I2 |
| I4 | Flutter框架 | 项目骨架 + ApiService + AppStateNotifier | I3 |
| I5 | 首页核心交互 | HomePage可运行：完成/换一个/卡住完整流程 | I4 |
| I6 | Project管理 | 创建/切换/进展页完整流程 | I5 |
| I7 | 体验打磨 | 动画/低能量/中断恢复/边界状态 | I6 |

## Iteration 1：基础设施

### 目标
建立所有后续层的共同基础：数据库、配置、数据模型。

---

### 设计确认

在开始编码前，对以下设计点做最终确认：

- [ ] 确认 `app_state` 单行表方案（row id=1 永远只有一行）
- [ ] 确认 Block 进度增量常量值（Bootstrap +20 / Push +15 / Wrap +10）
- [ ] 确认 `sessions` 中断阈值默认为 2 小时
- [ ] 确认项目上限 3 个由应用层（service）而非数据库层（UNIQUE index）强制

## Iteration 2：Step生成引擎

### 目标
实现 Agent 核心：格式验证、LLM工具调用、pattern选择、重试与fallback。
这是系统最核心的部分，独立于HTTP层，可纯单元测试。

---

### 设计确认

- [ ] 确认验证器黑名单词表（当前8个词，是否需要补充？）
- [ ] 确认 LLM 使用 Claude claude-sonnet-4-6（生产）；测试环境 mock 掉 LLM 调用
- [ ] 确认 fallback 模板库最少覆盖：4个 step_type × 2个 energy_mode = 8条
- [ ] 确认 pattern 轮换规则：连续2次同pattern被拒 → 轮换顺序（Light→Continuation→Completion→Decomposition→Recovery→Refinement→循环）

---

### 开发

```
backend/agent/
├── validators.py
│     validate_step_format(description: str) → ValidationResult
│       Pass1: 黑名单检查（8个模糊动词）
│       Pass2: 结构检查（动词1-4字 + 对象≥4字，总长8-30字）
│       返回：{ valid, error, verb, object_text }
│
├── prompt_builder.py
│     build_generation_prompt(block, pattern, energy_mode, avoid_list) → str
│     build_stuck_prompt(block) → str        # Decomposition专用
│     build_recovery_prompt(block) → str     # Recovery专用
│     build_light_prompt(block) → str        # Light专用
│
├── tools.py
│     定义6个工具的 Python 函数 + Anthropic tool schema（JSON）：
│       get_project_context, select_target_block, get_recent_rejections,
│       generate_step_candidate, validate_step_format, save_generated_step
│
└── agent.py
      select_pattern(trigger, recent_rejections) → StepPattern  # 纯确定性
      run(trigger, project_id, energy_mode, block_id=None) → StepOutput
        1. select_pattern()
        2-7. tool-calling loop（最多2次 validate 重试）
        fallback: 从 FALLBACK_STEPS[step_type][energy_mode] 取固定step
```

**Fallback模板库（内联于 agent.py）**
```python
FALLBACK_STEPS = {
    ("Bootstrap", "normal"): ("写第一个结构块的开头一句话", 10),
    ("Bootstrap", "low"):    ("给第一个结构块起一个标题", 5),
    ("Push",      "normal"): ("给当前块补充一个要点", 10),
    ("Push",      "low"):    ("改当前块的一个词让它更准确", 5),
    ("Wrap",      "normal"): ("检查当前块是否超过三行并删减", 10),
    ("Wrap",      "low"):    ("读一遍当前块，标记一个需要改的地方", 5),
    ("Transfer",  "normal"): ("写下一个结构块的第一句话", 10),
    ("Transfer",  "low"):    ("给下一个结构块起一个标题", 5),
}
```

## Iteration 3：后端API层

### 目标
所有 repositories、services、routers 完成，FastAPI 可用 httpx 测试客户端端到端验证。

---

### 设计确认

- [ ] 确认 `POST /state/session` 在每次 App 打开时调用（不是每次启动进程）
- [ ] 确认 `DELETE /projects/{id}` 是软删除（status→archived），不影响关联steps
- [ ] 确认 `/steps/complete` 响应中 `feedback_message` 格式：`"{block_title}已推进"`
- [ ] 确认中断检测：gap > 2h → trigger=resume；gap ≤ 2h → trigger=auto（不是新建session时跳过）

---

### 开发

**开发顺序：repo → service → router → main.py**

```
backend/repositories/
  project_repo.py   create, get, list_active, update_progress, archive
  block_repo.py     create_bulk, get_by_project, advance_progress, update_status
  step_repo.py      create, get_current, mark_completed, mark_status
  state_repo.py     get_state, set_active_project, set_energy_mode,
                    create_session, end_session, get_last_session

backend/services/
  project_service.py
    create_project(title, completed_block_ids) → StepResponse
      - 创建project + 4个固定blocks
      - 根据 completed_block_ids 设置对应block为completed
      - 调用 step_service.generate_next(trigger='auto')
    archive_project(id)
    recalculate_progress(project_id) → int  # avg of block progress_pct

  step_service.py
    generate_next(project_id, trigger, block_id=None) → StepResponse
      - 调用 agent.run()
      - 持久化step
      - 构造 StepResponse
    complete_step(step_id) → StepResponse
      - mark_completed + advance_progress + recalc + generate_next(trigger='complete')
    skip_step(step_id) → StepResponse
      - mark_status('skipped') + diversity_service.record + generate_next(trigger='skip')
    stuck_on_step(step_id) → StepResponse
      - mark_status('stuck') + generate_next(trigger='stuck', same block_id)

  session_service.py
    start_session() → trigger  # 检测中断，返回 'resume' or 'auto'
    end_session(session_id)

  diversity_service.py
    record_rejection(step_id, verb, object_text, pattern)
      - insert step_rejections
      - 保留最近3条（删除多余的）
    get_recent(project_id, limit=3) → list[StepRejection]

backend/routers/
  state.py    GET /state, PATCH /state/energy, POST /state/session
  projects.py GET /projects, POST /projects, GET /projects/{id},
              PATCH /projects/{id}/active, DELETE /projects/{id}
  steps.py    GET /steps/current, POST /steps/start,
              POST /steps/complete, POST /steps/skip, POST /steps/stuck
  blocks.py   GET /projects/{id}/blocks

backend/main.py
  app = FastAPI()
  lifespan: init_db() on startup
  include_router × 4
  CORS: allow localhost (Flutter dev)
```

对照 `doc/UI.md §3.1–3.3` 确认以下细节：
